# 微信云开发数据库权限配置指南

## 简化方案：所有集合设为"所有人可读写"

### 为什么这样做？

1. **微信云开发的安全规则功能有限**，不支持"仅管理端可写"这种精细控制
2. **固件和小程序都通过云函数中转**，云函数内部可以做权限校验
3. **简单可靠**，不容易出错

---

## 需要配置的集合

| 集合名 | 小程序权限 | 固件权限 | 说明 |
|--------|-----------|---------|------|
| `data_points` | 读写 | 读写（通过云函数） | 存储实时数据 |
| `device_commands` | 写 | 读（通过云函数） | 校准命令中转 |
| `device_status` | 读 | 写（通过云函数） | 设备状态上报 |

---

## 配置步骤（每个集合都要配置）

### 方法一：微信开发者工具（推荐）

1. 打开**微信开发者工具**
2. 点击顶部 **"云开发"** 按钮
3. 在云开发控制台，选择 **"数据库"**
4. 点击集合名称（比如 `data_points`）
5. 点击 **"权限设置"**
6. 选择 **"自定义安全规则"**
7. 粘贴以下规则：
   ```json
   {
     "read": true,
     "write": true
   }
   ```
8. 点击 **"保存"**
9. 重复以上步骤，配置所有集合（`data_points`、`device_commands`、`device_status`）

### 方法二：小程序代码里配置（不推荐）

在小程序代码里调用 `db.collection().add()` 时，如果权限不足会报错。需要在云端配置。

---

## 验证权限是否生效

配置完成后，可以在小程序里测试：

```javascript
// 测试读取
wx.cloud.database().collection('data_points').limit(1).get()
  .then(res => console.log('读取成功', res))
  .catch(err => console.error('读取失败', err));

// 测试写入
wx.cloud.database().collection('device_commands').add({
  data: {
    device_id: 'test',
    command: 'record_relax',
    status: 'pending',
    created_at: Date.now()
  }
}).then(res => console.log('写入成功', res))
  .catch(err => console.error('写入失败', err));
```

如果两条都成功，说明权限配置正确。

## 生产环境安全建议

当前方案（所有人可读写）在开发阶段没问题，但生产环境需要加强安全：

### 方案 A：配置 AppID 白名单

在云函数里检查请求的 `appid` 是否是自己的小程序：

```javascript
// 云函数入口
exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext();
  if (wxContext.ENV !== 'cloud1-d4gqmimmo05b12c94') {
    return { code: 403, msg: 'forbidden' };
  }
  // ... 正常逻辑
};
```

### 方案 B：使用云开发的"未登录模式" + "安全规则"

限制只有自己的小程序可以访问（需要配置 `openapi` 权限）。
