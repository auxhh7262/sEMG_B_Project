## 1. 项目概述

### 1.1 架构方案

微信云开发方案（v2.0.0），核心链路：

```
固件 → HTTP POST → 云函数(dataIngest) → 云数据库(data_points)
                                                  ↓
                                       小程序 wx.cloud.database().watch()
                                                  ↓
                                       calibrate / realtime / analysis 展示
```

- 固件通过 7 个云函数 HTTP 触发器与云端通信
- 小程序通过 `wx.cloud.callFunction()` 调用云函数
- 实时数据通过云数据库 `watch()` 接收
- BLE 仅用于配网阶段（WiFi SSID/密码写入）

### 1.2 页面结构

| Tab | 页面 | 功能 |
|-----|------|------|
| 第 1 个 | calibrate | 肌电校准（放松 + 收缩两阶段） |
| 第 2 个 | realtime | 实时监测数据展示 |
| 第 3 个 | analysis | 历史数据分析 + 图表 + CSV 导出 |
| 第 4 个 | network | BLE 配网 + 云端设备状态查询 |

### 1.3 通用通信方式

所有页面不再直连固件 WebSocket，统一通过云开发通信：

- **命令下发：** `wx.cloud.callFunction({ name: 'sendDeviceCommand' })` → 云端 `device_commands` 集合 → 固件轮询
- **实时数据：** `wx.cloud.database().watch()` 监听 `data_points` 集合
- **校准状态：** 从 `localStorage` 读取 `calib_data` 快照（EEPROM 为权威源）

## 2. calibrate — 肌电校准页面

**TabBar 位置：** 第 1 个 Tab

### 2.1 功能概述

- 用户 Profile 录入（姓名/性别/年龄/利手）
- 两阶段校准（relax 10s → active 15s）
- 实时显示 RMS/MDF
- 校验结果并保存到云端 + localStorage

### 2.2 状态机

```
idle ──(onStartCalibration)──► relax
                                │ (收到云端结果)
                                ▼
                           active_ready
                                │ (onStartActive)
                                ▼
                           active_contract
                                │ (收到云端结果)
                                ▼
                              result
                                │ (onConfirmResult)
                                ▼
                               saved
```

| 阶段 | 含义 | 用户操作 |
|------|------|---------|
| idle | 待机 | 点击"开始校准" |
| relax | 放松采集 10s | 保持放松 |
| active_ready | relax 完成 | 准备最大收缩 |
| active_contract | 最大收缩 15s | 全力握拳 |
| result | 显示结果 | 确认保存或重试 |
| saved | 已保存 | 切换到实时监测 |

### 2.3 核心流程

```
onLoad:
  _initDeviceId()           — 从 storage 读取 deviceId
  _loadUserProfile()        — 从 storage 读取 current_user
  _loadCalibData()          — 从 storage 读取 calib_data

onStartCalibration:
  1. 检查用户 Profile → 无则弹表单
  2. 检查 deviceId → 无则提示配网
  3. setData phase='relax'
  4. _sendCommand('record_relax')
  5. _startPolling() → 每 1s 轮询云端状态

relax → active_ready:
  收到 sessions 含 calibration.relax_rms
  显示实时 RMS/MDF 数据

onStartActive → active:
  _sendCommand('record_active')
  倒计时 15s，实时显示 RMS/MDF

active → result:
  收到 sessions 含 calibration.active_rms
  _validateResult():
    rms_ok = activeRms > relaxRms × 2.0 && activeRms >= 0.5mV
    mdf_ok = relaxMdf ∈ [10,250] && activeMdf ∈ [10,250]

onConfirmResult → _doSaveCalib:
  _sendCommand('save_calib', { name, age, gender, handedness })
  wx.setStorageSync('calib_data', 结果)
  setData({ saved: true })
```

### 2.4 云端命令流程

```
小程序 → 云端 device_commands:
  { device_id, command, params, status: 'pending', created_at }

固件轮询 (每 10s) → getDeviceCommand API → 执行 → ackDeviceCommand

小程序轮询 (每 1s) → 云端 sessions 集合 → 获取 calibration 数据
```

### 2.5 数据结构

```javascript
Page.data = {
  phase: 'idle',               // idle|relax|active_ready|active|result|saved
  connected: false,             // 云开发连接状态
  currentUser: null,            // 用户 Profile
  showUserForm: false,          // 用户表单弹窗
  deviceId: '',
  liveRelaxRms: null,          // 放松阶段实时 RMS
  liveRelaxMdf: null,          // 放松阶段实时 MDF
  liveActiveRms: null,         // 收缩阶段实时 RMS
  liveActiveMdf: null,         // 收缩阶段实时 MDF
  relaxRms: null,              // 最终 relax RMS
  relaxMdf: null,              // 最终 relax MDF
  activeRms: null,             // 最终 active RMS
  activeMdf: null,             // 最终 active MDF
  endMdf: null,                // 收缩阶段结束 MDF
  validation: null,            // { ok, rms_ok, mdf_ok }
  saved: false
}
```

## 3. realtime — 实时监测页面

**TabBar 位置：** 第 2 个 Tab

### 3.1 功能概述

- 从云数据库 watch 实时接收肌电数据
- 5 行历史数据滚动表格
- 连接状态与校准状态显示

### 3.2 页面生命周期

```
onLoad:
  _startWatch() → 启动云数据库 watch

onShow:
  加载校准状态（localStorage）
  如 watch 未启动则重新启动

onHide:
  只跳过渲染（_tabVisible = false），不关闭 watch

onUnload:
  _stopWatch()
```

### 3.3 数据接收

**云数据库 watch：** `db.collection('data_points').orderBy('timestamp', 'desc').limit(1).watch()`

**数据点格式：**

| 字段 | 编码 | 显示值 |
|------|------|--------|
| rms | 原始值 | /1000 = mV |
| activation | 0~1000 | /10 = 0~100% |
| mdf | 0~2500 | /10 = Hz |
| fatigue | 0~1000 | /10 = 0~100% |
| quality | 0~100 | 直接显示 % |

**渲染限频：** 每 500ms 一次 `setData`，避免高频渲染卡死。

### 3.4 数据结构

```javascript
Page.data = {
  connected: false,             // 云数据库 watch 状态
  quality: '--',                // 信号质量 %
  timeStr: '--:--:--',         // 当前时间
  historyRows: [],              // 最多 5 行
  algorithm: '无校准',          // 校准状态
  scrollIntoId: 'row-0',
}
```

**historyRow：** `{ time, rms, act, mdf, fat, q }` — 新数据在头部（unshift），满 5 行 pop 尾部。

### 3.5 校准状态检测

```javascript
const c = wx.getStorageSync('calib_data')
algorithm = (c && c.relax_rms > 0) ? '已校准' : '无校准'
```

### 3.6 固件端对应

| 小程序端 | 固件端 |
|---------|--------|
| `_startWatch()` | `NetManager::pushDataPoint()` |
| `_onDataPoint(pt)` | `dataIngest` 云函数 |

## 4. analysis — 数据分析页面

**TabBar 位置：** 第 3 个 Tab

### 4.1 功能概述

- 通过云函数查询历史疲劳度数据
- 预置日期范围 + 自定义时间选择（含时分）
- 疲劳度趋势图（Canvas 2D，渐变色曲线 + 三色阈值区域）
- 汇总分析卡（峰值疲劳度、峰值时间、监测时长、状态、趋势）
- 疲劳等级统计（正常/轻度/重度条数+占比）
- CSV 导出（游标分页拉取全部数据 → 发送到微信）

### 4.2 核心流程

```
选择日期范围 → onQuery:
  _loadData():
    1. 在前端计算 startTs/endTs（毫秒时间戳），传递给云函数
    2. 调用云函数 queryDataPoints({ startDate, endDate, startTs, endTs })
    3. 云函数返回最多 2000 条降采样数据 + 聚合统计
    4. 计算摘要、结论、趋势
    5. 绘制疲劳度趋势图
    6. 显示疲劳等级统计信息
```

### 4.3 日期范围

用户通过 `<picker>` 选择预设或自定义时间范围：

| 预设 | 范围 |
|------|------|
| 昨天 | 昨天 00:00:00 ~ 昨天 23:59:59 |
| 最近 3 天 | 前 3 天 00:00:00 ~ 昨天 23:59:59（不含今天） |
| 最近 7 天 | 前 7 天 00:00:00 ~ 昨天 23:59:59（不含今天） |
| 最近 30 天 | 前 30 天 00:00:00 ~ 昨天 23:59:59（不含今天） |
| 自定义 | 支持选择起止日期 + 起止时分（HH:mm） |

**关键规则：** 预设范围（昨天/3天/7天/30天）均**排除今天**，避免实时数据影响统计准确性。

**自定义模式：** 使用两个 `<picker mode="date">` 和两个 `<picker mode="time">`，分别选择开始/结束的日期和时刻。

**时间戳计算：** 所有模式都在前端（手机端，UTC+8）计算 `startTs`/`endTs` 毫秒时间戳，直接传递给云函数，避免时区解析偏差。

### 4.4 汇总分析卡

查询完成后显示一张汇总卡，包含以下信息：

| 项目 | 说明 |
|------|------|
| 峰值疲劳度 | 区间内最高疲劳度百分比 |
| 峰值时间 | 峰值出现的时间（M/DD HH:mm） |
| 监测时长 | 首末数据时间差（<1分钟 / N分钟 / Nh Nm） |
| 状态 | 🟢状态正常 / 🟡轻度疲劳 / 🔴严重疲劳 / ⚪数据不足 |
| 趋势标签 | 稳定波动 / 缓慢上升 / 持续上升 / 缓慢下降 / 明显下降 |
| 趋势描述 | 文字说明趋势含义 |
| 建议 | 基于状态给出运动建议 |

**状态判断逻辑（_calcConclusion）：**

| 条件 | 状态 |
|------|------|
| 时长 < 2 分钟 | 数据不足 |
| maxFatigue < 30% | 状态正常 |
| 30% ≤ maxFatigue < 70% | 轻度疲劳 |
| maxFatigue ≥ 70% | 严重疲劳 |

**趋势判断逻辑（_calcTrend）：**

比较前后两半数据的平均疲劳度差值：

| 差值 | 趋势 |
|------|------|
| \|diffPct\| < 10% | 稳定波动 |
| 10% < diffPct ≤ 20% | 缓慢上升 |
| diffPct > 20% | 持续上升 |
| -20% ≤ diffPct < -10% | 缓慢下降 |
| diffPct < -20% | 明显下降 |

### 4.5 疲劳等级统计

云函数聚合查询原始数据，返回各疲劳等级的真实数量：

| 等级 | 条件 | 含义 |
|------|------|------|
| 正常（good） | fatigue < 30% | 肌肉状态正常 |
| 轻度疲劳（warn） | 30% ≤ fatigue < 70% | 轻度疲劳 |
| 重度疲劳（danger） | fatigue ≥ 70% | 重度疲劳 |

统计使用云函数 `aggregate().group()` 基于**原始数据**计算，非降采样数据，确保百分比准确。

### 4.6 Canvas 绘图（疲劳度趋势图）

- 单图表：疲劳度趋势（渐变色曲线）
- 三色背景区域：绿色 0-30%、橙色 30-70%、红色 70-100%
- 网格线、Y 轴刻度（100 在上，0 在下）、X 轴时间标签
- **X 轴两行显示**：第一行 HH:mm:ss.ms，第二行 M/DD
- 时间标签精确到毫秒
- X 轴起止时间使用云函数返回的 `firstTs`/`lastTs`（数据库实际首末数据时间），非降采样数据
- 疲劳度阈值线：30%（绿色虚线）、70%（红色虚线）
- 峰值标注：标记疲劳度最高点，带数值标签
- 渐变色描边：下绿→中橙→上红
- DPR 适配高清屏
- 数据不足 2 点时显示"数据不足，无法绘制"

### 4.7 CSV 导出

**导出流程：**

```
1. 点击"导出原始数据"按钮
2. 前端计算 startTs/endTs 毫秒时间戳，调用 queryDataPoints(export: true)
3. 云函数游标分页（每批 3000 条），循环拉取全部数据
4. 进度提示："正在导出 X/Y 条数据..."
5. 生成 CSV 文件（UTF-8 编码）
6. 隐藏"导出原始数据"按钮，显示"发送到微信"按钮
7. 点击"发送到微信" → wx.shareFileMessage() 拉起微信分享面板
```

**CSV 格式（6 列，含 MDF 原始数据）：**

| 列 | 格式 | 示例 |
|----|------|------|
| Timestamp | YYYY-MM-DD HH:mm:ss.ms | 2026-07-01 00:00:00.123 |
| RMS(uV) | 保留 3 位小数 | 0.852 |
| Activation(%) | 保留 1 位小数 | 80.0 |
| MDF(Hz) | 保留 1 位小数 | 84.9 |
| Fatigue(%) | 保留 1 位小数 | 22.3 |
| Quality | 整数 | 85 |

**文件名规则：**
- 非自定义模式：`sEMG_{startDate}_{endDate}.csv`（如 `sEMG_2026-06-30_2026-06-30.csv`）
- 自定义模式：`sEMG_{startDate}_{stFmt}-{endDate}_{etFmt}.csv`（如 `sEMG_2026-07-01_0000-0021.csv`）

### 4.8 云函数调用

```
queryDataPoints({ startDate, endDate, startTs, endTs, export, pageSize, cursorTs })
```

| 字段 | 说明 |
|------|------|
| startDate/endDate | 日期字符串（YYYY-MM-DD） |
| startTs/endTs | 毫秒时间戳（前端计算，优先于日期字符串） |
| export | false=查询模式（降采样2000条+聚合统计），true=导出模式（游标分页） |
| pageSize | 导出模式每批大小（默认 3000） |
| cursorTs | 导出模式上一批最后一条的 timestamp，用于游标下一页 |

**返回数据：**

查询模式：
```javascript
{
  code: 0, data: [...],           // 降采样数据
  total: 202610,                   // 原始数据总数
  downsampled: true,               // 是否降采样
  firstTs: 1782807192846,          // 首条数据时间戳
  lastTs: 1782835198765,           // 末条数据时间戳
  goodCount: 180000,               // 正常数量（原始数据聚合）
  warnCount: 20000,                // 轻度疲劳数量（原始数据聚合）
  dangerCount: 2610,               // 重度疲劳数量（原始数据聚合）
}
```

导出模式：
```javascript
{
  code: 0, data: [...],            // 本批数据
  total: 202610,                   // 总数（仅首次返回）
  hasMore: true,                   // 是否还有下一页
  pageSize: 3000,
}
```

## 5. network — 网络配置页面

**TabBar 位置：** 第 4 个 Tab

### 5.1 功能概述

- BLE 扫描并连接 sEMG 设备
- 读取设备真实 ID
- 写入 WiFi SSID / 密码
- 接收配网结果通知
- 从云端查询设备状态

### 5.2 BLE 服务与特征值

| UUID | 权限 | 用途 |
|------|------|------|
| 19B10000-... | Service | sEMG BLE 服务 |
| 19B10001-... | Write | WiFi SSID |
| 19B10004-... | Write | WiFi Password |
| 19B10005-... | Read/Notify | Device ID（v3.0） |
| 19B10006-... | Notify | Provision Result（v3.0 新增） |

### 5.3 状态管理

| 主状态 | 说明 |
|--------|------|
| idle | 初始未连接 |
| scanning | BLE 扫描中（10s 超时） |
| connecting | BLE 连接中 |
| configuring | 写入 SSID + 密码 |
| connected | 配网成功，已获 IP |
| failed | 配网失败 |

### 5.4 配网流程

```
1. startScan() — BLE 扫描 sEMG 设备（10s 超时）
2. _connectBle() — 连接 BLE
3. _readDeviceId() — 读取 19B10005 特征（设备真实 ID）
4. notify 订阅 19B10006 — 接收配网结果
5. 写入 SSID (19B10001) + Password (19B10004)
6. 接收 notify:
   - CONNECTING → 等待 WiFi 连接（35s 超时）
   - OK → 从云端获取 IP → 配网成功
   - FAIL → 配网失败
7. _getStatusFromCloud() — 从云端查询设备 IP/SSID
```

### 5.5 云端 API

| 端点 | 用途 |
|------|------|
| `POST /getDeviceStatus` | 获取设备 IP / SSID |
| `POST /sendDeviceCommand` | 发送命令（如 reset_wifi） |

### 5.6 重置配网

三种方式：
1. **BLE 已连接：** 写入 `"RESET_WIFI"` 到 SSID 特征
2. **仅知 deviceId：** 调用云端 `sendDeviceCommand` 发送 `reset_wifi`
3. **都失败：** 提示"未发现设备"

### 5.7 数据结构

```javascript
Page.data = {
  connected: false,         // BLE 连接状态
  configured: false,        // 配网完成
  deviceId: '',
  ip: '',                   // 从云端获取
  ssid: '',
  status: 'idle',           // idle|scanning|connecting|configuring|connected|failed
  provisionStep: '',        // 配网进度
}
```

### 5.8 固件端对应

| 小程序操作 | 固件函数 |
|-----------|---------|
| `_readDeviceId()` | `BleConfigServer::onRead()` |
| 写入 SSID | `BleConfigServer::onWrite()` |
| 写入 PASS → 启动连接 | `NetManager::onWiFiStarted()` |
| notify CONNECTING | `NetManager::onWiFiConnected()` |
| notify OK + IP | `NetManager::reportStatus()` |
| notify FAIL | `NetManager::onWiFiFailed()` |
| 重置配网 | `handleResetWifi()` |
