## 3. 创建数据库集合

在云开发控制台 → 数据库，手动创建以下 **5 个集合**：

| 集合名 | 用途 |
|--------|------|
| `data_points` | 肌电数据点（固件上传，小程序读取） |
| `devices` | 设备注册信息 |
| `device_commands` | 设备命令队列（小程序写入，固件轮询） |
| `device_status` | 设备状态上报（IP/SSID） |
| `calibration_results` | 校准结果 |

**操作步骤（重复 5 次）：**

1. 云开发控制台 → 数据库 → 点击 **"+"**
2. 输入集合名称 → 点击 **"确定"**
3. 点击集合右侧 **"权限设置"**（见下节）
4. 选择 **"仅创建者可写，所有人可读"**
5. 点击 **"保存"**

### 可选：添加索引

| 集合 | 索引字段 | 用途 |
|------|---------|------|
| `data_points` | `device_id`(升) + `timestamp`(降) | 按设备查历史数据 |
| `devices` | `device_id`(升, 唯一) | 设备去重 |
| `device_commands` | `device_id`(升) + `status`(升) | 查待执行命令 |
| `device_status` | `device_id`(升) + `timestamp`(降) | 查设备最新状态 |

> 首次调用云函数时会尝试自动创建集合，但权限仍需手动设置。

## 4. 设置集合权限

**⚠️ 关键：** 每个集合必须设为 **"仅创建者可写，所有人可读"**。

| 选项 | 效果 | 选择 |
|------|------|------|
| 仅创建者可读写 | ❌ 小程序无法读取 | ❌ |
| **仅创建者可写，所有人可读** | ✅ 固件(后端)写入，小程序读取 | ✅ |
| 所有人可读写 | ⚠️ 开发阶段可用，生产环境不安全 | ⚠️ |
| 自定义安全规则 | 需额外配置 | 可选 |

**开发阶段简化方案（全部可读写）：**

```json
{ "read": true, "write": true }
```

**注意：** 固件通过云函数写入数据库，不是直接写集合。所以集合的 "创建者" 是云函数调用方，不是固件本身。

## 5. 配置 HTTP 触发器

以下 **5 个云函数** 需要开启 HTTP 访问，供固件调用：

| 云函数 | 固件是否调用 | HTTP 触发 URL |
|--------|-------------|---------------|
| `deviceRegister` | ✅ 是 | `https://cloud1-d4gqmimmo05b12c94.service.tcloudbase.com/deviceRegister` |
| `dataIngest` | ✅ 是 | `https://cloud1-d4gqmimmo05b12c94.service.tcloudbase.com/dataIngest` |
| `getDeviceCommand` | ✅ 是 | `https://cloud1-d4gqmimmo05b12c94.service.tcloudbase.com/getDeviceCommand` |
| `ackDeviceCommand` | ✅ 是 | `https://cloud1-d4gqmimmo05b12c94.service.tcloudbase.com/ackDeviceCommand` |
| `reportDeviceStatus` | ✅ 是 | `https://cloud1-d4gqmimmo05b12c94.service.tcloudbase.com/reportDeviceStatus` |

**配置步骤：**

1. 云开发控制台 → 云函数 → 选择函数
2. 点击 **"HTTP 访问"** 标签
3. 点击 **"开启 HTTP 访问"**
4. 复制生成的 URL
5. 重复以上步骤配置 5 个函数

固件中 URL 基地址在 `NetManager.cpp` 中配置：

```cpp
const char* CLOUD_BASE_URL = "https://cloud1-d4gqmimmo05b12c94.service.tcloudbase.com";
```

## 9. 常见问题

### Q1: 找不到"云开发"按钮？

确保 `project.config.json` 中有 `cloudfunctionRoot` 配置，微信开发者工具版本 >= 1.02。

### Q2: 部署失败，提示"缺少 config.json"

确保每个云函数目录都有 `config.json` 文件。

### Q3: 权限设置后小程序仍无法读取数据？

检查：
- 权限是否设置为 **"仅创建者可写，所有人可读"**（不是"仅创建者可读写"）
- 小程序是否调用了 `wx.cloud.init({ env: 'cloud1-d4gqmimmo05b12c94' })`

### Q4: HTTP 访问返回 404？

确认 HTTP 触发器已开启，URL 格式正确：`https://{env-id}.service.tcloudbase.com/{functionName}`

### Q5: 固件无法连接云端？

排查：
1. 检查固件 WiFi 是否已连接（串口日志）
2. 检查云函数 HTTP 触发器是否已开启
3. 确认固件中 `CLOUD_BASE_URL` 与云开发控制台一致
